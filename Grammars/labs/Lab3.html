Lexical Analyzer

Your last program is to build a lexical analyzer (token recognizer) for a language of your choice.  The language must be a Turing complete language, i.e., it must have a conditional construct of some sort (if, switch, etc.) and a looping construct of some sort (while, do-while, repeat, etc.).

Your task is to identify the tokens for the language and find out rules for defining those tokens.  For instance, Identifiers in C++ are language tokens used to identify variables, functions, methods or objects.  Identifiers in C++ must start with a letter or an underscore and can contain upper and lowercase letters, digits and underscores.  Other token types include:

  • Object methods and properties.
  • Operators
  • Keywords and reserved words
  • Symbols
  • Constants
The task of a lexical analyzer  is to recognize legal tokens and identify errors in tokens. It doesn't need to check for syntax errors.

Input and Output

  1. Create at least 3 test files in the language of your choice.
  2. Read an input file and print out all tokens.
  3. Print out appropriate error message for illegal tokens.
